# news-digest

Многоисточниковый дайджест ИИ-новостей для Claude Code — извлекает, резюмирует и изучает последние технологические новости ИИ из 11 источников.

```
/news-digest                    # полный дайджест + слой обучения
/news-digest --no-learn         # только новости (быстрый режим)
/news-digest llm agent          # фильтрация по ключевому слову
/news-digest --sources hn,hf    # только определённые источники
```

---

## Принцип работы

**Шаг 1 — Уведомление и загрузка**: выводит баннер с оценкой времени (20–40 с для новостей, +60–80 с со слоем обучения), затем загружает данные из 11 источников в одном вызове Bash:

| Источник | Содержимое |
|----------|-----------|
| `hn` | Главная страница Hacker News |
| `arxiv` | Новейшие статьи arXiv cs.AI |
| `hf` | HuggingFace Daily Papers (курированный ИИ) |
| `github` | Трендовые репозитории GitHub |
| `anthropic` | Официальные новости Anthropic |
| `openai` | Новости OpenAI (через Jina Reader, обходит Cloudflare) |
| `reddit` | r/MachineLearning + r/LocalLLaMA + r/agents |
| `langchain` | RSS блога LangChain |
| `github_watch` | Релизы отслеживаемых репозиториев (openclaw, opencode…) |
| `openclaw` | RSS блога OpenClaw |
| `clawhub` | Последние обновления навыков ClawHub |

**Шаг 0 — Определение профиля проекта**: автоматически определяет текущий каталог проекта и загружает соответствующий профиль контента. Встроенные профили:

| Проект | Фокус | Приоритетные источники |
|--------|-------|----------------------|
| `cc_manager` | Claude Code harness, оркестрация агентов, MCP, проектирование инструментов | anthropic, hn, github, langchain, clawhub |
| `thinking_of_memory` | Академические статьи, методы анализа данных, статистическое моделирование | arxiv, hf, reddit |

Пользовательские профили можно добавить в `~/.claude/news-digest-profiles.json`:
```json
{
  "my_project": {
    "display": "my-project",
    "focus": "описание области интересов",
    "extra_keywords": ["keyword1", "keyword2"],
    "preferred_sources": ["arxiv", "hn"],
    "learner_instruction": "Как news-learner должен ориентировать анализ для этого проекта..."
  }
}
```

Если ни один профиль не подошёл, используется режим по умолчанию без фильтрующего смещения.

**Шаг 2 — Фильтрация и дедупликация** по ключевому слову, авторитетности источника и схожести заголовков. Ключевые слова и веса источников профиля применяются на этом шаге. Если более 3 источников не вернули данные, вверху показывается предупреждение о частичном сбое `[Диагностика]`; если более 50% не работают — alert `⚠️ [Критическое предупреждение]` с шагами по устранению неполадок.

**Шаг 3 — Вывод** структурированного CLI-дайджеста:

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  AI News Digest  |  2026-03-22  |  23 записи
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

### Hacker News
1. [Заголовок]
   Резюме · ссылка

### HuggingFace Daily Papers
1. [Название статьи]
   Выдержка из аннотации · ссылка
...

Источники: hn arxiv hf github anthropic  |  Сбои: openai  |  Фильтр: agent mcp
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**Шаг 4 — Слой обучения** (опционально, ~60–80 с): агент `news-learner` анализирует релевантные для вашего ИИ-рабочего процесса элементы, сравнивает их с существующими Claude Code skills/agents/patterns и выдаёт приоритизированные рекомендации:

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  Возможности для обучения  |  2 элемента проанализировано
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

[Adopt] Инструмент X — реальный пробел в уровне оркестрации
  → Шаг 1: создать новый skill на основе этого дизайна
  → ссылка: https://...

[Learn] Фреймворк Y — пересекается с существующим паттерном Z
  → архивировано в ~/.claude/projects/<project>/memory/tech-watch.md
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

Уровни рекомендаций: `[Priority Adopt]` → `[Adopt]` → `[Learn]` → `[Skip]`

- **`[Adopt]` / `[Priority Adopt]`**: появляется интерактивный запрос решения; введите номер для действия, `s N` для пропуска, `w N` для понижения до `[Learn]`
- **`[Learn]`**: дедуплицируется по URL и архивируется в `tech-watch.md` в каталоге памяти текущего проекта
- **`[Skip]`**: отмечается в сессии, файл не записывается

## Использование

```
/news-digest [topics...] [--sources СПИСОК] [--limit N] [--no-learn]
```

| Аргумент | Описание | По умолчанию |
|----------|----------|-------------|
| `topics` | Фильтр по ключевым словам — через пробел, логика ИЛИ | все элементы |
| `--sources` | ID источников через запятую (см. таблицу выше) | все 11 источников |
| `--limit N` | Макс. элементов на источник | 5 |
| `--no-learn` | Пропустить слой обучения, выводить только новости (~60–80 с экономии) | выкл. |
| `--channel` | Канал вывода (`cli` доступен; `slack`/`email`/`file` запланированы) | `cli` |

**Примеры:**

```bash
/news-digest                             # полный запуск, все источники
/news-digest --no-learn                  # быстро — только новости, без агента
/news-digest llm agent mcp               # фильтр по ключевым словам
/news-digest --sources hn,arxiv,hf       # только три источника
/news-digest --sources github --limit 10 # GitHub Trending топ-10
/news-digest mcp --sources anthropic,openai  # два официальных источника, фильтр MCP
```

## Реальный пример: от новостей к новой возможности

Реальный цикл принятия от 2026-03-24, демонстрирующий полный рабочий процесс `/news-digest` → `news-learner` → немедленное решение.

### Что запустило процесс

При запуске `/news-digest` без фильтров слой обучения обнаружил один элемент уровня `[Adopt]`:

> **"How we monitor internal coding agents for misalignment"** — OpenAI Engineering Blog
> *Фреймворк мониторинга поведения агентов кодирования в реальном времени: обнаружение уклонений, проверки переполнения области, обнаружение аномалий разрешений, измерение дрейфа намерений.*

`news-learner` сравнил его с существующими возможностями платформы и обнаружил **реальный пробел**:

```
Ключевая возможность: мониторинг поведения агента в реальном времени — обнаружение дрейфа намерений и уклонений
Текущее состояние:   skill-review выполняет только статический анализ определений; нет мониторинга в реальном времени
Рекомендация:  [Adopt]
План принятия:
  1. Создать агент agent-monitor: читать трассировку задачи, проверять по четырёхсигнальному чеклисту
  2. Создать паттерн agent-monitoring: направляет координаторов запускать мониторинг после высокорисковых задач
```

В конце дайджеста появился запрос решения:

```
⚡ Требуется действие (1 элемент):

  1. [Adopt] How we monitor internal coding agents for misalignment
     План: создать агент agent-monitor, читать трассировку задачи, обнаруживать дрейф намерений и уклонения

Введите 1 для немедленного действия, s 1 для пропуска, w 1 для понижения до [Learn]
```

### Что было создано (пользователь ввёл `1`)

Три файла созданы в той же сессии:

**`~/.claude/agents/agent-monitor.md`** — читает файл трассировки завершённой задачи и проверяет четыре категории сигналов (уклонение, переполнение области, аномалия разрешений, дрейф намерений), записывает ранжированные предупреждения в `memory/agent-alerts.md`:

| Уровень | Значение | Действие координатора |
|---------|----------|----------------------|
| ✅ CLEAN | Норма | Продолжить |
| ⚠️ WATCH | Незначительный сигнал, возможный ложноположительный | Зафиксировать, отслеживать тренд |
| 🚨 ALERT | Высокорисковый сигнал | Приостановить, уведомить пользователя |
| 🛑 BLOCK | Подтверждённые вредоносные индикаторы | Остановить + откат |

**`~/.claude/patterns/agent-monitoring.md`** — руководство координатора: какие задачи требуют мониторинга после выполнения и как вызывать `agent-monitor`.

**`~/.claude/hooks/post_bash_error.sh`** — пассивный слушатель, срабатывающий при каждом сбое вызова Bash, записывает структурированные записи об ошибках в `memory/errors.md` и отмечает высокорисковые паттерны (curl не из белого списка, `rm -rf`, force-push, чтение чувствительных файлов) в `memory/agent-alerts.md`.

### Почему это важно

Полный цикл — обнаружение → анализ пробела → решение → создание — завершился за одну сессию. `news-learner` выявил возможность, которой платформе действительно не хватало (мониторинг в реальном времени против статического анализа), и новый агент был готов к работе, как только пользователь ввёл `1`.

Это задуманный рабочий процесс: `/news-digest` выявляет то, что стоит знать; `news-learner` сопоставляет это с вашими реальными пробелами; запрос решения делает принятие беспрепятственным.

## Реальный пример: обнаружение обновления версии модели в процессе тестирования конвейера

Второй реальный случай от 2026-04-05, возникший во время Stage 3 поведенческой оценки при выполнении `/skill-test packer/news-digest` — демонстрирует, как `news-learner` обнаруживает актуальный для платформы релиз и выдаёт конкретную рекомендацию по обновлению.

### Что послужило триггером

В ходе eval-прогона Stage 3 `/news-digest` получил данные из официального новостного источника Anthropic:

> **«claude-sonnet-4.6 теперь доступен»** — официальные новости Anthropic
> *Anthropic выпустил claude-sonnet-4.6 (ID модели: `claude-sonnet-4-6`), новейшее поколение Sonnet с улучшенными возможностями рассуждения и использования инструментов. Предыдущее поколение claude-sonnet-4.5 остаётся доступным, но больше не является рекомендованной версией по умолчанию для новых развёртываний.*

`news-learner` проверил установленных агентов и навыки платформы и обнаружил захардкоженные ссылки на `sonnet`, указывающие на предыдущее поколение:

```
Ключевая возможность: актуальность версии модели — используют ли агенты/навыки последний рекомендованный Sonnet
Текущее состояние: news-learner.md использует `model: sonnet` (голый псевдоним, разрешается рантаймом CC;
                   версия не зафиксирована, намеренность не задокументирована)
Рекомендация:  [Adopt]
План принятия:
  1. Решение: зафиксировать явный ID версии или сохранить голый псевдоним с документированным намерением
  2. При фиксации: обновить поле model на `claude-sonnet-4-6` в затронутых файлах агентов
  3. При сохранении псевдонима: добавить комментарий, подтверждающий намеренное плавающее поведение
```

В конце дайджеста появился запрос решения:

```
⚡ Требуется действие (1 элемент):

  1. [Adopt] Выпущен claude-sonnet-4.6 — news-learner всё ещё использует голый псевдоним `sonnet`
     План: зафиксировать claude-sonnet-4-6 или задокументировать псевдоним как намеренный

Введите 1 для немедленного действия, s 1 для пропуска, w 1 для понижения до [Learn]
```

### Что произошло (пользователь ввёл `1`)

Голый псевдоним `model: sonnet` был намеренным выбором — CC разрешает его до последнего Sonnet при установке, обеспечивая «автообновление» без изменения файлов. Сессия прояснила это намерение и задокументировала его inline:

```yaml
# agents/news-learner.md (frontmatter)
model: sonnet   # намеренный голый псевдоним — разрешается до последнего Sonnet при установке CC
```

Содержимое файла не было существенно изменено; решение было зафиксировано как намеренный no-op с обоснованием.

### Почему это важно

Данный случай иллюстрирует другой класс элементов `[Adopt]`: не пробел в возможностях, а **решение о позиции обслуживания**. `news-learner` обнаружил релиз в день его выхода; владелец платформы должен был решить — фиксировать или оставить плавающим — вместо того чтобы пассивно обнаруживать дрейф спустя недели.

Контекст конвейера (Stage 3 eval) повысил ценность обнаружения по сравнению с обычным запуском дайджеста: поле модели тестируемого агента было напрямую задействовано, превратив фоновое уведомление о релизе в конкретный пункт для проверки.

## Разработка

### Evals

`evals/evals.json` содержит 6 тестовых случаев, охватывающих разбор аргументов, загрузку данных и слой обучения:

| ID | Сценарий | Что проверяется |
|----|----------|----------------|
| 1 | `/news-digest` (полный запуск, без аргументов) | Загружает все 11 источников, выводит дайджест, слой обучения выполняется |
| 2 | `/news-digest --no-learn` | Пропускает агент news-learner; выводит только сводку новостей |
| 3 | `/news-digest llm agent` (фильтр по ключевым словам) | Только элементы с `llm` или `agent` в заголовке/резюме |
| 4 | `/news-digest --sources hn,hf --limit 3` | Загружает только указанные источники, макс. 3 элемента каждый |
| 5 | Путь MCP/Jina (переменная окружения `JINA_API_KEY` присутствует) | Инструмент Jina MCP используется предпочтительно для загрузки контента |
| 6 | Недопустимый ID источника | Выводит сообщение об ошибке с допустимыми ID; загрузка не производится |

Ручное тестирование (в сессии Claude Code):
```bash
/news-digest --no-learn     # быстрая проверка, eval 2
/news-digest llm            # фильтр по ключевым словам, eval 3
```

Запуск всех evals с помощью eval loop от skill-creator (если установлен):
```bash
python ~/.claude/skills/skill-creator/scripts/run_loop.py \
  --skill-path ~/.claude/commands/news-digest.md \
  --evals-path evals/evals.json
```
